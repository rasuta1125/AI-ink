## ✨ 主要機能

### 🎯 コア機能（3+1ボタン）
1. **Hookメーカー**: 引きの1行 ×5案
   - 5系統（数字・質問・逆説・ギャップ・事実）各1案
   - 24-48字目安、語尾連続回避

2. **CTAビルダー**: 行動喚起 ×5案  
   - 動詞＋場所＋期限の構造化
   - 35-70字、次アクション明確

3. **#タグ6**: ハッシュタグ6個
   - 汎用2・カテゴリ2・ニッチ2の配合
   - 正規化・重複除去済み

4. **フル本文**: 完全版 ×3案
   - Hook→本文→CTA→#6の構成
   - 短文(120-180字)・中文(200-350字)・長文(400-600字)

## 📋 使い方

### 基本操作
1. **コンテンツ設定**: 目標・業種・トーン・トピックを入力
2. **機能選択**: 4つのボタンから必要な機能をクリック  
3. **結果取得**: 生成されたコンテンツをワンクリックでコピー
4. **SNS投稿**: そのままSNSやブログで使用

### 推奨ワークフロー

3. **フル生成**: 最後に統合版で完成形を確認

## 🏗️ 技術仕様

### アーキテクチャ
```
[フロントエンド] → [Honoサーバー] → [OpenAI GPT API]
     UI              API Routes         AI生成 + フォールバック
```

### API仕様
```bash
# Hookメーカー
POST /api/hooks
Request: {"goal":"認知","industry":"salon","tone":"フレンドリー","topic":"空き枠案内"}
Response: {"hooks":["hook1",...], "meta":{...}}

# CTAビルダー
POST /api/ctas
Request: {"goal":"CV","path":"プロフィール","deadline":"今週中","topic":"初回相談"}
Response: {"ctas":["cta1",...], "meta":{...}}

# ハッシュタグ
POST /api/hashtags6  
Request: {"industry":"ec","topic":"新作商品"}
Response: {"hashtags":["#tag1",...], "meta":{...}}

# フル本文
POST /api/captions
Request: {"goal":"CV","industry":"salon","tone":"フレンドリー","topic":"空き枠","length":"short"}
Response: {"captions":[{"text":"...","parts":{...}}], "meta":{...}}
```

### 入力パラメータ
- `goal`: 認知 | 保存 | CV
- `industry`: creator | salon | ec | local | other  
- `tone`: フレンドリー | 専門家 | エモい | きっぱり
- `topic`: トピック（100字以内）
- `path`: プロフィール | リンク | DM | フォーム
- `deadline`: なし | 今週中 | 今月中
- `length`: short | mid | long
